# Информационная система электронного расписания занятий

## Описание проекта

Курсовая работа по дисциплине "Технология разработки программного обеспечения". Тема: Информационная система электронного расписания занятий.

Веб-приложение для управления расписанием занятий в образовательном учреждении (колледж, техникум, вуз).

## Функциональность

- Создание и редактирование расписания занятий
- Управление справочниками:
  - Преподаватели
  - Аудитории
  - Дисциплины
  - Учебные группы
- Предотвращение пересечений по аудиториям
- Предотвращение пересечений по преподавателям
- Просмотр расписания с фильтрацией по группам
- Система авторизации с двумя ролями:
  - **Администратор** - полный доступ (добавление/удаление расписания, управление справочниками)
  - **Пользователь** - только просмотр расписания

## Технологии

- Python 3
- Flask (веб-фреймворк)
- Flask-Login (система авторизации)
- PostgreSQL (база данных)
- SQLAlchemy (ORM)
- HTML/CSS (интерфейс)

## Требования

- Python 3.8+
- PostgreSQL 12+
- pip (менеджер пакетов Python)

## Установка

1. Клонируйте репозиторий:
```
bash
git clone <URL_репозитория>
cd kurs
```

2. Установите зависимости:
```
bash
pip install -r requirements.txt
```

3. Настройте подключение к базе данных в файле `.env`:
```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=schedule_db
DB_USER=postgres
DB_PASSWORD=123
```

4. Создайте базу данных в PostgreSQL:
```
sql
CREATE DATABASE schedule_db;
```

5. Создайте таблицы в базе данных:
```
bash
python create_db.py
```

## Запуск

Запустите приложение:
```bash
python app.py
```

Откройте в браузере: http://127.0.0.1:5000

## Авторизация

После первого запуска перейдите по адресу `/init-db` для создания тестовых пользователей:

- **Администратор**: логин `admin`, пароль `admin123`
- **Пользователь**: логин `user`, пароль `user123`

Администратор имеет полный доступ к системе. Обычные пользователи могут только просматривать расписание.

## Использование

1. Перейдите по адресу http://127.0.0.1:5000
2. Войдите в систему (или просматривайте расписание без входа)
3. Добавьте данные в справочники (требуется вход администратора):
   - Преподаватели (ФИО, должность)
   - Аудитории (номер, корпус, вместимость)
   - Дисциплины (название)
   - Группы (название, специальность)
4. Создайте расписание занятий
5. При добавлении занятий система автоматически проверяет пересечения
